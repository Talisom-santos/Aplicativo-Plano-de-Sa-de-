# Aplicativo-Plano-de-Saude

# 🚀 App de Saúde & Bem-estar
Este projeto consiste no desenvolvimento de um wireframe mobile de baixa fidelidade focado na jornada de agendamento de consultas psicológicas. O desafio foi equilibrar a complexidade de um app de plano de saúde com a necessidade de um fluxo acolhedor e sem fricção.

# 📌 Contexto e Problema
Usuários de planos de saúde muitas vezes enfrentam interfaces sobrecarregadas com burocracias (boletos, carteirinhas, exames).
O desafio: Criar um fluxo onde um usuário emocionalmente vulnerável consiga agendar uma terapia com o mínimo de esforço cognitivo.

# 🧠 Estratégia de UX
Persona Alvo
Perfil: Adulto, rotina ocupada.

Estado Emocional: Vulnerável, baixa tolerância a erros ou processos longos.

Necessidade: Rapidez, acolhimento e clareza.

Arquitetura da Informação
A estrutura foi pensada para priorizar a Saúde Mental como um serviço de destaque, isolado das funções administrativas comuns.

Home: Porta de entrada com gatilho emocional positivo.

Busca/Filtro: Redução de opções para evitar a "paralisia de escolha".

Seleção de Horário: Interface visual limpa e direta.

Confirmação: Feedback imediato e positivo.

# 📱 O Wireframe (Estrutura)
1. Tela Inicial (Area de login)

UX Writing: Uso de frases como "Entrar ?" para Inicializar o acesso.

2. Fluxo de Agendamento
Filtros Rápidos: Seleção de atendimento logo no topo.

Cards de Profissionais: Informações essenciais (abordagem, CRP e disponibilidade) visíveis sem necessidade de clique extra.

3. Finalização
Confirmação: Tela limpa com resumo do agendamento (Data, Hora e Profissional).

CTA Secundária: "Adicionar ao calendário" para evitar esquecimentos.

# 🛠️ Ferramentas Utilizadas
Figma: Prototipagem de baixa fidelidade.

Metodologia: User-Centered Design (UCD).

# 🧪 Plano de Validação
Para validar a eficácia do protótipo, propõe-se:

Teste de 5 Segundos: Verificar se o botão de ajuda emocional é identificado imediatamente.

Task Success Rate: Medir quantos usuários completam o agendamento sem retornar telas.
